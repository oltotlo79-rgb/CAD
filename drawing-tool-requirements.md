# 2D製図ツール 要件定義書（ドラフト v0.2）

> 壁打ちで合意した内容をまとめたもの。Claude Code への実装指示の土台として使う。
> 末尾の「未確定事項（TODO）」は、実装に入りながら詰める。
> v0.2: レビューで見つかった矛盾を解消（補助線の2種類化、保存方式、PDF出力方針、編集ツール追加、グリッド段階、線の数値編集ルール、表題欄の自動反映）。

---

## 1. 目的・背景

- 工作物を製作する前の三面図作成にかかる工数を削減する。
- 3D CADで立体図を描いてから三面図を起こすワークフローを、Excel感覚で手早く線を引ける軽量な2D製図ツールで置き換える。
- 対象は主に機械部品の三面図（正面図・平面図・側面図）。

---

## 2. 対象環境・配布

- 単一HTMLファイルとして動作する（HTML / CSS / JavaScript、外部依存は最小限）。
- ブラウザで開くだけで動く。インストール不要。
- 将来ファイル関連付け（ダブルクリックで図面を開く）等のOS連携が欲しくなった場合は、同じコードをTauriで包んでデスクトップアプリ化できる（作り直し不要）。今回は開発サイクルが最短の単一HTMLを選択。
- 複数のWindows PCで使う。ファイルをコピーするだけで配布できること。
- 保存は File System Access API による上書き保存を基本とし、非対応ブラウザではダウンロード方式にフォールバックする（詳細は §12）。
- オフラインで動作する（ネット接続不要）。

---

## 3. アーキテクチャ方針

- 描画は HTML Canvas を使用。図形は「データオブジェクト」として保持し、状態変化のたびに再描画する。
- **すべての座標・寸法は実寸mmで保持**する。縮尺と画面ズームは描画変換にのみ適用し、データ自体には掛けない。
- ビルド不要が望ましい。ライブラリを使う場合も単一HTMLに収める。

---

## 4. 単位・座標系・縮尺

- 単位は mm 固定。角度は度（°）。
- 内部座標は用紙基準の絶対mm。y軸は上向き（左下原点系）。
- **ユーザー原点**を任意位置に設定できる。座標の入力・表示はユーザー原点からの相対値になる。図形データは絶対座標で保持するため、原点を移動しても既存の図形は動かず、数値の見え方だけが変わる。
- 初期原点は図面枠の内側・左下。
- 縮尺は図面全体で1つ。プリセット（1:1 / 1:2 / 1:2.5 / 1:5 / 1:10 / 1:20 / 1:50 / 2:1 / 5:1 / 10:1）＋任意数値入力。
- 入力は実寸で行う。縮尺を後から変更しても図面データは壊れず、表示が縮む/拡大するだけ。寸法値も実寸で自動表示される。

---

## 5. グリッドとスナップ

### グリッド
- 実寸mm基準。10 / 5 / 2 / 1 / 0.5 / 0.1 mm。
  （5mm・2mmはズーム自動切替を滑らかにするための中間段階。10→1mmが10倍飛びだと、表示px範囲に収まるグリッドが存在しないズーム域が生じるため）
- ズーム連動で自動切替。画面上のグリッド間隔が一定px範囲（目安 8〜80px）に収まるよう、mm間隔を段階的に切り替える。ズームアウトで粗く、ズームインで細かく。
- 手動固定モードも用意する。
- 基準グリッド（10mm）を濃く、補助グリッドを薄く表示。

### スナップ（各種ON/OFF可）
- グリッドスナップ
- オブジェクトスナップ：**端点・中点・円の中心/四半点・要素同士の交点**
- 投影補助線・45°ミラー線・それらの交点へのスナップ
- スナップ対象が近いときはハイライト表示する。

---

## 6. 作図要素（スコープ：フル機能。実装は段階的）

### 形状
直線、連続線／ポリライン、矩形、円、円弧、楕円、スプライン、点

### 線種（JIS）
外形線（太実線）、かくれ線（破線）、中心線（一点鎖線）、寸法・引出線（細実線）、想像線（二点鎖線）、ハッチング（断面の斜線）
- 線の太さは JIS 標準（0.25 / 0.35 / 0.5 / 0.7 mm 等）を線種に割り当て、画面・印刷の両方に反映する。

### 注記・寸法
文字・注記、寸法記入、引出線、幾何公差、表面粗さ記号、溶接記号、部品番号バルーン

### 図面管理
図面枠、表題欄、部品表（BOM）

---

## 7. 作図・編集操作

- 作図はクリック式・ドラッグの両対応。
- 数値入力作図：始点座標・長さ・角度を数値で指定でき、すべて相互に連動する。
- 選択：単一クリック、範囲選択。
- 編集：移動・複製・回転・削除・**ミラー（反転）・トリム／延長・オフセット（平行複製）**。
  - フィレット（角R）・面取りは拡張フェーズ。
- **Undo / Redo（必須）**。
- 選択要素のプロパティ（座標・長さ・角度・半径など）を数値で編集できる。
- **線の数値編集の基準点ルール：常に始点基準**。線は「始点＋長さ＋角度」で決まるものとして扱う。
  - 長さ・角度を数値で変更 → 始点固定のまま終点が動く。
  - 始点座標を数値で変更 → 長さ・角度を保ったまま線全体が平行移動する。
  - 終点座標を数値で変更 → 始点固定のまま終点が動く（長さ・角度が再計算される）。

---

## 8. 補助線（2種類）

補助線は性質の異なる2種類を持つ。混同しないこと。

### 8a. 投影補助線（第三角法・一時ガイド）
- 自動化レベルは「**補助線が出るだけ**」。図形同士は連動しない。**一時ガイドであり、エンティティとしては保存しない。印刷もしない。**
- トリガー：**選択した要素**の端点・特徴点から水平/垂直の延長線を表示する。
- 45°ミラー線：オン/オフできるガイド。平面図↔側面図の奥行き転写に使う。
- 補助線・その交点・45°線との交点にスナップできる（手で合わせる精度を担保する要）。

### 8b. 作図補助線（下書き線・保存されるエンティティ）
- ユーザーが自分で引く下書き線。**通常のエンティティとして保存され、`aux`（補助線）レイヤーに載る。**
- 「下書き線を引いてから外形線をなぞる」運用に対応する。
- `aux` レイヤーの印刷をオフにすることで、下書き線を消したクリーンな印刷ができる（§10）。
- スナップ対象になる。

---

## 9. 寸法記入

- 種類：長さ（水平・垂直・平行）、直径 φ、半径 R、角度、面取り C。
- 寸法値は実寸から**自動計測**。手動上書きも可（現物合わせ等）。
- JIS準拠：寸法線・寸法補助線・矢印。単位mmは省略表示、角度は°。
- 配置：2点指定、または要素を選んで自動配置。
- 公差：機能として持つが**デフォルトはオフ**。必要なときに ±両側 または 上下個別（例 `50±0.1` / `50 +0.1/0`）を付与できる。

---

## 10. レイヤー

- 用途別プリセットレイヤー：外形線 / かくれ線 / 中心線 / 寸法 / 注記 / 補助線（作図補助線用）。
- 各要素をレイヤーに割り当てる。
- レイヤーごとに表示 ON/OFF、印刷 ON/OFF。
- 補助線レイヤー（§8b の作図補助線が載る）を印刷オフにすることで、下書き線を消したクリーンな印刷ができる。§8a の投影補助線は一時ガイドでありレイヤーには載らない。
- 任意レイヤーの追加は拡張フェーズ。
- **レイヤーのデータ構造は Phase 1 から存在させる**（全エンティティが layer 属性を持つため）。レイヤーの UI（パネル・表示/印刷 ON/OFF 切替）は Phase 4 で実装する。

---

## 11. 図面枠・表題欄

- 用紙サイズ：A2 / A3 / A4 / B3 / B4 / B5。縦・横を選択可能。
- 図面枠を自動描画。
- 表題欄は**カスタム可能**。デフォルトで一般的な項目（図番・図名・尺度・投影法・材質・作成者・日付など）を用意し、項目の追加・削除・名称変更・配置調整ができる。
- **尺度・用紙サイズは図面設定からの自動反映（バインド項目）とし、手入力不可**。縮尺変更時に表題欄が古いまま残る不整合を構造的に防ぐ。日付・作成者などは手入力。
- 表題欄の `position` は**用紙左下原点の絶対mm座標**で保持する（図面枠内側基準ではない）。

---

## 12. ファイル形式・保存

- ネイティブ保存：独自JSON形式。続きから編集できる。
- **保存方式：上書き保存＋フォールバック**
  - Chrome / Edge：File System Access API により、開いたファイルへ**そのまま上書き保存**（Ctrl+S の感覚）。
  - 非対応ブラウザ：ダウンロード方式に自動フォールバック。
- **ファイルを開く方法**：ファイル選択ダイアログに加え、**JSONファイルのウィンドウへのドラッグ&ドロップ**でも開ける（ダブルクリック起動ができない不便の緩和）。
- **未保存のままタブを閉じようとしたら警告**を出す（軽量な安全網）。
- **クラッシュ復元用の自動スナップショット**：編集中の図面を一定間隔（または操作の区切り）で IndexedDB に自動保存する。ブラウザやPCが落ちた後にアプリを開き直すと、未保存の変更がある場合に復元候補を提示する。
  - これはファイルへの自動保存ではない（ファイルはあくまで手動保存のみ）。ブラウザ内の一時領域に持つ安全網であり、正式な保存は上書き保存/ダウンロードで行う。

---

## 13. 印刷・出力

- 一次エクスポート形式：**SVG**（用紙実寸のmm単位で出力。他ツールでも扱いやすい）。
- **PDF化はブラウザの印刷ダイアログ経由**（「PDFに保存」）とする。印刷用ビューを用紙実寸で組み、**倍率100%（「実際のサイズ」）指定**で実寸を担保する。専用PDFライブラリ＋日本語フォント埋め込みは行わない（単一HTMLの肥大化を避ける）。
- 印刷用ビューでは印刷OFFレイヤー・投影補助線を除外する。
- 1ファイル = 1図面（1枚）。三面図・断面図・部品表はその1枚に自由配置する。

## 15. 実装ロードマップ（順序は調整前提）

> **実装状況(2026-07-03)**: Phase 1〜4・6 実装済み(表題欄含む)。未実装: Phase 5(ハッチング・部品表・バルーン)、Phase 7(幾何公差・表面粗さ・溶接記号)、Phase 8(スプライン・フィレット/面取り編集)、角度寸法、公差入力UI(寸法値の自由上書きで代用可)。

- **Phase 1 — 土台**：座標系・縮尺・グリッド（自動切替）・用紙枠・ユーザー原点・JSON保存/読込（上書き保存＋フォールバック）・直線・連続線・矩形
  （レイヤーはデータ構造のみ持つ。UIは Phase 4）
- **Phase 2 — 形状と線種**：円・円弧・楕円、線種一式（想像線＝二点鎖線を含む）、文字・注記、オブジェクトスナップ
- **Phase 3 — 寸法・注記**：寸法記入（長さ / φ / R / 角度 / 面取り）、公差、引出線
- **Phase 4 — 編集強化・投影補助・レイヤー**：ミラー・トリム／延長・オフセット、投影補助線・45°ミラー線、作図補助線（§8b）、レイヤーUI
- **Phase 5 — 断面・組立**：ハッチング、部品表・バルーン
- **Phase 6 — 出力**：SVG エクスポート、印刷（PDF化）の実寸保証
- **Phase 7 — 記号系**：幾何公差・表面粗さ・溶接記号
- **Phase 8 — 拡張**：スプライン、フィレット（角R）・面取り編集 ほか

※ スプラインは実装が軽ければ前倒し可能。
